# Документация для Разработчиков (Developer Guide)

Данный документ описывает внутреннее устройство проекта **Sravni Tool**. Он предназначен для разработчиков, которые хотят поддерживать, изменять или расширять функционал приложения.

Проект состоит из двух полностью независимых версий, выполняющих одну и ту же бизнес-логику:
1. **Desktop-версия** (`Sravni.py` на Python + Tkinter)
2. **Web-версия** (`index.html` на HTML/CSS/Vanilla JS)

---

## 1. Desktop-версия (Python / Tkinter)

### 1.1 Архитектура и UI
Приложение построено на базе встроенной библиотеки `tkinter`. 
- Интерфейс не использует `.ui` файлы или сторонние фреймворки вроде PyQt, весь UI собирается кодом.
- Основной лэйаут построен на `Frame` и `pack(side="left")` / `pack(side="right")`.
- **`create_text_block()`**: Фабричная функция для создания текстовых редакторов. Она упаковывает сам `Text`, скроллбар и холст номеров строк (`LineNumbers`) в единый `Frame`, чтобы они не разъезжались.

### 1.2 Нумерация строк (`LineNumbers`)
Класс `LineNumbers(tk.Canvas)` отвечает за рисование номеров.
- **Механика**: Он привязывается к скроллу и событиям редактирования основного `Text`.
- Функция `redraw()` проходит циклом по видимым строкам (`dlineinfo()`) и рисует текст на Canvas по координате `y` соответствующей строки. Это гарантирует 100% точность, в отличие от простого подсчета `\n`.

### 1.3 Работа буфера обмена и горячих клавиш (ВАЖНЫЙ ХАК)
В Windows 11 с Python 3.13 возникают проблемы с перехватом `Ctrl+C / Ctrl+V` при использовании кириллической раскладки: Tcl/Tk выбрасывает исключение `bad event type or keysym`.
- **Решение**: В функции `apply_hotkeys()` мы перебираем не только `v` и `c`, но и их русские аналоги из системы X11 (`Cyrillic_em`, `Cyrillic_es`).
- Привязка обернута в `try-except TclError`, чтобы на ОС без поддержки этих ключей программа не падала.
- **Вставка (`tk_textPaste`)**: Мы не используем генерирование события `<<Paste>>`, мы вызываем напрямую команду Tcl/Tk `tk_textPaste` (через `.tk.call('tk_textPaste', widget._w)`). Это железобетонный метод, который работает всегда.

### 1.4 Сборка в EXE
Мы используем `PyInstaller`.
Команда для сборки: `python -m PyInstaller --onefile --noconsole --icon="icon.ico" --name "Sravni" Sravni.py`
Файл `Sravni.spec` генерируется автоматически. При внесении изменений в код достаточно пересобрать проект этой командой, удалив старые папки `build` и `dist`.

## 2. Web-версия (HTML / CSS / JavaScript)

### 2.1 Архитектура
Проект сознательно написан как **Single-file Application** (все стили и скрипты внутри `index.html`) без использования React, Vue или сборщиков вроде Webpack.
- **Цель**: Максимально быстрый старт, возможность использовать локально без веб-сервера.
- Используются современные CSS-фичи: CSS Variables (`:root`), Flexbox, Grid (`.panes`).

### 2.2 Нумерация строк и верстка
- Окна имеют **фиксированную высоту** (`height: 350px;`), чтобы интерфейс не «прыгал» при вставке 10 000 строк.
- Для нумерации используется отдельный `div` (`.line-numbers`), который прокручивается синхронно с полем `textarea`.
- **Функция `syncScroll(id)`**: Привязывается к атрибуту `onscroll` редактора. Она просто берет `scrollTop` текстового поля и присваивает его колонке с номерами.
- Важно: `line-height` и `font-size` у `textarea` и `.line-numbers` заданы пиксель-в-пиксель, иначе при прокрутке возникнет рассинхрон.

### 2.3 Бизнес-логика (Поиск потерь)
Функция `findLosses()` работает по алгоритму со сложностью **O(N + M)**, где N и M — количество строк:
1. Очистка строк с помощью `trim()` и удаление пустых.
2. Массив окна 2 переводится в нижний регистр и складывается в структуру `Set` (множество). Хэш-таблица `Set` обеспечивает поиск элемента за O(1).
3. Проходимся одним циклом по массиву окна 1. Если элемента нет в `Set`, добавляем в `losses`.

### 2.4 Работа с буфером обмена (Clipboard API)
Приложение использует современный `navigator.clipboard.writeText()` (асинхронный метод).
Для визуального отклика добавлена CSS-анимация `pulse`, которая применяется через класс `copied` в JS `setTimeout`.

## Памятка по внесению изменений (Roadmap)

### Если вы меняете логику сравнения:
Вам придется обновить две функции:
- `find_losses()` в `Sravni.py` (используется Python `set()`).
- `findLosses()` в `index.html` (используется JS `Set()`).

### Если вы добавляете новую кнопку:
1. **Python**: добавьте новый `create_btn()` в блок `btn_frame` (в `Sravni.py`). Выберите цвет из словаря HEX-кодов (например, `#17A2B8` для инфо-действий).
2. **HTML**: добавьте новую кнопку `<button class="btn btn-X">` в блок `.toolbar`. Возьмите подходящую SVG-иконку, например, с сайта *Feather Icons* или *Lucide*.

### Развертывание
Любые PUSH в ветку `main` на GitHub автоматически обновляют Web-версию через **GitHub Pages**. (Обычно занимает 30-60 секунд). 
Для Desktop-версии вам нужно вручную запустить PyInstaller и загрузить новый `.exe` в раздел Releases.
